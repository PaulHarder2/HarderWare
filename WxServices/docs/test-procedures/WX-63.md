# WX-63 Manual Test Procedure — Containerized WxMonitor.Svc

**What this verifies:** `WxMonitor.Svc` runs correctly as a Linux Docker container
(`services/docker-compose.yml`) instead of a Windows service — it builds, reaches the
host SQL Server, starts, writes its log and heartbeat through the bind mount where
WxManager reads them, and exports telemetry to the collector. This is the first-run
proof for the flagged unknowns (bind-mount write permissions under non-root UID 1000,
log4net rolling over the 9p mount, `host.docker.internal` DB reach). The build/pattern
pieces (image builds, the `WxPaths` env seam, the banner) are build-/unit-covered; this
document is the verification record for the live container run.

**Recording results:** when every section passes, set the ticket's custom fields:
**Test Proc** = `docs/test-procedures/WX-63.md` and **Test Result** =
`PASS yyyy-mm-dd @ <version>`. If anything fails, set
`FAIL yyyy-mm-dd @ <version> — section N` and describe it in a ticket comment.

"**Expect:**" lines are pass/fail criteria — if reality differs, note the section and
continue; the list of failed sections is the run's deliverable.

---

## 0. Setup / precondition

1. **Stand down the Windows service so the two do not collide** (they would share the
   log, heartbeat, DB, and alert email). In an **elevated PowerShell**:

   ```powershell
   Stop-Service -Name WxMonitorSvc
   Set-Service  -Name WxMonitorSvc -StartupType Disabled
   ```

   **Expect:** `Get-Service WxMonitorSvc` shows `Stopped`, StartType `Disabled`.
2. Confirm Docker Desktop is up and the real (gitignored) config is in place:

   ```powershell
   docker info | Select-String 'Server Version'
   Test-Path C:\Code\HarderWare\services\wxmonitor\appsettings.local.json
   ```

   **Expect:** a server version prints; the path test returns `True`.

## 1. Build + start the container

```powershell
cd C:\Code\HarderWare\services
docker compose up -d --build wxmonitor
docker compose ps
```

**Expect:** the image builds and the container starts with no error; `docker compose ps`
shows `wxmonitor` as `running`/`Up`.

## 2. "Application started" banner (reached the DB and started)

```powershell
docker compose logs wxmonitor | Select-String 'starting\.','Database ready','Application started'
```

**Expect:** the log shows `WxMonitor.Svc 1.51.0 starting.` (version present, and **no**
`(commit …)` fragment — the WX-63 banner change), `Database ready.`, and
`Application started. Press Ctrl+C to shut down.`. The banner proves
`EnsureSchemaAsync` succeeded against `host.docker.internal,1433`.

## 3. Log through the bind mount + worker running (UID 1000 can write; WxManager sees the log)

```powershell
Get-Content C:\HarderWare\Logs\wxmonitor-svc.log -Tail 8
```

**Expect:** `wxmonitor-svc.log` accumulates fresh lines from the container — proving log4net
writes through the 9p bind mount under non-root UID 1000 — including a recent
`Monitor cycle complete` / `Next monitor check in 5 minute(s)`, which proves the worker is
actually running. (WxMonitor does **not** write its own heartbeat file: it *reads* the other
services' heartbeats and has no self-watchdog — "WxMonitor does not watch itself," DESIGN.md §10.)

## 4. Telemetry to the collector / Grafana

Prerequisite: telemetry must be enabled — `Telemetry:Enabled=true` (the shared config sets this,
and the container's `appsettings.local.json` sets it explicitly) and the container's
`Telemetry:OtlpEndpoint` ends in `/v1/metrics`. **If telemetry is disabled, skip this section.**
Open Grafana (`http://localhost:3000`) and the OTel service dashboard.

**Expect:** metrics from the `wxmonitor` container appear (it reaches the collector at
`host.docker.internal:4318`). A short delay after startup is normal.

## Cleanup / cutover note

If the run **passes**, the container may stay up as the new WxMonitor (the Windows service
stays disabled as a fallback; run `sc.exe delete WxMonitorSvc` only once the container is
proven over a few days). If the run **fails**, revert to the Windows service:

```powershell
cd C:\Code\HarderWare\services; docker compose down
Set-Service -Name WxMonitorSvc -StartupType Automatic; Start-Service -Name WxMonitorSvc   # elevated
```
