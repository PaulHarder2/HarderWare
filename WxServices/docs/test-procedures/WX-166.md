# WX-166 — Languages registry + supported-language gate + WxManager Languages tab

Manual test procedure (WxManager is WPF and not CI-built; the migration + report
read-path need verification against a real database and live cycles). Run against a
**local/dev `WeatherData` database — never production** until WX-160 has left IN TEST
(see the WX-166 deploy guardrail). Record the outcome in the ticket's **Test Result**
field as `PASS yyyy-mm-dd @ 1.27.0` or `FAIL …`.

## 0. Prerequisites

1. `dotnet build WxServices.sln` is clean (WPF projects included).
   *Expect:* 0 errors.
2. Apply the migration to the dev database (run from the `WxServices` repo root, where the
   `src/` paths resolve; on Windows add `--msbuildprojectextensionspath 'C:\HarderWare\BuildCache\WxServices\MetarParser.Data\obj'`):
   `dotnet ef database update --project src/MetarParser.Data/MetarParser.Data.csproj --connection "<dev WeatherData connection>"`.
   *Expect:* `AddLanguagesTableAndRecipientFk` applies without error.

## 1. Migration + seed

1. Query `SELECT COUNT(*) FROM Languages;`.
   *Expect:* **184** rows (full ISO 639-1).
2. Query `SELECT IsoCode, DisplayName, IsEnabled FROM Languages WHERE IsEnabled = 1 ORDER BY IsoCode;`.
   *Expect:* exactly two rows — `en / English / 1` and `es / Spanish / 1`.
3. Query `SELECT RecipientId, LanguageId FROM Recipients;` and cross-check against the
   pre-migration `Language` values.
   *Expect:* every recipient that had `"English"` now points at the `en` row, `"Spanish"`
   at the `es` row, and `NULL`/other values are `NULL` (no data loss; no orphan FK). The
   old `Language` column no longer exists.

## 2. WxManager Languages tab

1. Launch WxManager, complete Setup, open the **Languages** tab.
   *Expect:* left "All languages" list shows the disabled languages (filterable); right
   "Supported" list shows **English** and **Spanish** only.
2. Select **French** (or any non-en/es) on the left and click **Enable ▶**.
   *Expect:* refused with a status message that no report templates exist for it yet
   (WX-167); it stays on the left, `IsEnabled` unchanged.
3. Type in the filter box.
   *Expect:* the left list narrows by name/ISO code.
4. With at least one recipient assigned Spanish (see §3), select **Spanish** on the right
   and click **◀ Disable**.
   *Expect:* refused with a message naming the count of assigned recipients and pointing
   to the Recipients tab; Spanish stays enabled.
5. Reassign/remove that recipient's Spanish assignment, then disable Spanish.
   *Expect:* Spanish moves to the left list; `IsEnabled = 0` persists. Re-enable it
   (templates exist) to restore state.

## 3. Recipient editor

1. Open the **Recipients** tab and select an existing recipient.
   *Expect:* the **Language** field is a dropdown (not a text box); it shows the
   recipient's language, or **"(service default)"** when `LanguageId` is null.
2. Open the dropdown.
   *Expect:* only **enabled** languages appear, plus the **"(service default)"** entry at
   the top. Disabled languages are absent.
3. Set a recipient to **Spanish** and Save; reopen the recipient.
   *Expect:* the selection persists; `SELECT LanguageId FROM Recipients WHERE RecipientId='…'`
   shows the `es` row id. Setting it to "(service default)" and saving stores `NULL`.
4. Create a **new** recipient.
   *Expect:* the Language field defaults to **"(service default)"**.

## 4. Report read-path (runtime — the In Test functional check)

1. With a recipient assigned **Spanish** and one on the **service default** (English),
   trigger a report cycle (scheduled or startup diagnostic) for their locality.
   *Expect (service log + delivered email):* the Spanish recipient's report body,
   `<html lang="es">`, and subject render in Spanish; the default/English recipient's in
   English. No recipient is silently downgraded to English. (This is the live confirmation
   that the FK → ISO-code read-path resolves correctly.)
2. Inspect the welcome path for a brand-new Spanish recipient.
   *Expect:* the welcome email renders in Spanish (greeting + schedule sentence).

## Notes

- The management UI is English-only by design; only recipient-facing reports localize.
- Enabling beyond `en`/`es` becomes possible once WX-167 supplies their templates.
