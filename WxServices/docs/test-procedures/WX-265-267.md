# WX-265 + WX-267 — functional verification (§13)

**Change under test (1.45.2):** the daypart vocabulary rename — `PartMorning`/`PartAfternoon`/
`PartEvening` → `DayPart2`/`DayPart3`/`DayPart4`, plus a new `DayPart1` ("early hours") for the
00:00–06:00 block — renamed in **both** `LanguageTemplates` and `PromptGlossaryTokens`, with the en
`DayPart1` row seeded. `DayPart1` ships **Required**, so the enabled target languages (es/eo/de) are
briefly missing it after deploy and WX-250 top-up backfills them one-per-cycle. WX-267 is the
bundled runbook (`docs/adding-a-new-report-token.md`), pure-docs.

**Deploy precondition (chosen, WX-265/WX-267):** deploy during a window with **no scheduled sends
pending** for the rest of the local day, so the transient suppression of es/eo/de while `DayPart1`
backfills has no recipient impact.

**Boundary:** the WxReportSvc deploy of **1.45.2** (from `deploy-history.log`; the verify script
resolves it via `deploy-info.sh`). All shell steps run in **bash (WSL)**; DB steps run in **Windows
PowerShell** (`sqlcmd` with `-E -C`). Log/DB timestamps are UTC.

## 2. Migration landed in BOTH tables (DB, PowerShell)

```powershell
sqlcmd -S localhost\SQLEXPRESS -d WeatherData -E -C -W -Q `
 "SET NOCOUNT ON;
  SELECT 'en LanguageTemplates' AS chk, Token, Phrase FROM LanguageTemplates lt
    JOIN Languages l ON l.Id = lt.LanguageId
    WHERE l.IsoCode='en' AND Token LIKE 'DayPart_' ORDER BY Token;
  SELECT 'old Part* remaining (must be 0)' AS chk, COUNT(*) AS n FROM LanguageTemplates
    WHERE Token IN ('PartMorning','PartAfternoon','PartEvening');
  SELECT 'PromptGlossaryTokens dayparts' AS chk, Token FROM PromptGlossaryTokens
    WHERE Token LIKE 'DayPart_' OR Token LIKE 'Part%' ORDER BY Token;"
```

- **Expect:** the en rows are `DayPart1`="early hours", `DayPart2`="Morning", `DayPart3`="Afternoon",
  `DayPart4`="Evening"; **old Part\* remaining = 0**; `PromptGlossaryTokens` shows `DayPart2`,
  `DayPart3`, `DayPart4` and **no** `Part*`.

## 3. Top-up backfills DayPart1 for every enabled target (DB, may take a few cycles)

```powershell
sqlcmd -S localhost\SQLEXPRESS -d WeatherData -E -C -W -Q `
 "SET NOCOUNT ON;
  SELECT l.IsoCode,
         CASE WHEN EXISTS (SELECT 1 FROM LanguageTemplates t
                           WHERE t.LanguageId=l.Id AND t.Token='DayPart1')
              THEN 'has DayPart1' ELSE 'MISSING (awaiting top-up)' END AS daypart1,
         t2.Phrase AS daypart1_phrase
  FROM Languages l
  LEFT JOIN LanguageTemplates t2 ON t2.LanguageId=l.Id AND t2.Token='DayPart1'
  WHERE l.IsEnabled=1 ORDER BY l.IsoCode;"
```

- **Expect (within ~15 min of deploy):** every enabled language shows `has DayPart1` with a
  non-empty phrase (en "early hours"; es/eo/de their generated term). Re-run until all are filled —
  one target fills per ~5-minute cycle.

## 4. Watch the fills and QA each phrase (the WX-267 operator loop)

Tail the log and watch the per-language fills:

```bash
tail -f /mnt/c/HarderWare/Logs/wxreport-svc.log | grep -E "WX-250: '|Template generation for '"
```

- **Expect:** lines like `WX-250: generating 1 missing token(s) for 'es' (Spanish)…` then
  `WX-250: 'es' topped up 1 token(s) — READY.`, one target per cycle.
- For each filled language, open **WxManager → Vocabulary**, select it, find the `DayPart1` row, and
  **verify the generated phrase reads correctly** for the pre-dawn 00:00–06:00 block (edit in place +
  Save if not). Read low-resource languages (eo) with extra suspicion.

## 5. Rendered output unchanged (WX-190 / A1)

Confirm a freshly rendered report still labels the 00–06 block by its **clock band**, not a word:

```powershell
sqlcmd -S localhost\SQLEXPRESS -d WeatherData -E -C -Q `
 "SET NOCOUNT ON; SELECT TOP 1 EmailBody FROM CommittedSends
   WHERE RecipientId='paul_en' ORDER BY Id DESC" -y 0
```

- **Expect:** in the Extended Forecast grid, the **Conditions cell of a day whose 00–06 block is
  still live** shows the clock-band label `00-06 - …` (e.g. `00-06 - Clear and dry`) — the 00–06
  block is labeled by its **clock range, not a daypart word**. Inspect that specific cell (not the
  whole email — a station name or other prose could legitimately contain a word like "early"
  elsewhere): it must **not** read "early hours" or `Overnight`. This is the WX-190 behavior,
  unchanged by adding `DayPart1` (A1: the token is seeded but not consumed until WX-264).
