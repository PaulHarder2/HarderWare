# WX-212 — Test Procedure

**Change:** The WxManager **Languages** tab now auto-refreshes while it is the visible tab — a
`DispatcherTimer` (10s) re-reads the registry only while the tab is shown (`Loaded` starts it +
reloads, `Unloaded` stops it; the default `TabControl` unloads/reloads tab content, so these bracket
visibility), so a language generating service-side flips `[Generating…] → [Ready]/[Blocked]` **in
place** without the operator switching tabs. The reload is single-flight (coalescing) and preserves
the current selection by ISO code across the rebind.

**Ships in:** WxManager (`LanguagesTab`), release **1.35.2**. WxManager is **not CI-built**.

**No `WX-212-verify.sh`:** this is a WxManager **UI** behaviour with no durable log/DB fingerprint
(the tab just re-queries and rebinds), so it is verified by direct observation — there is nothing a
log/DB script could assert. Test Result is recorded from the manual run below.

---

## 1. Build (pre-merge)

WxManager is excluded from CI, so build it explicitly and confirm it compiles clean:

```bash
dotnet build WxServices/WxServices.sln
```

**Expect:** build succeeds, 0 warnings (WxManager included).

## 2. Functional verification (manual, on the deployed WxManager)

1. Launch the deployed **WxManager** (1.35.2) and open the **Languages** tab.
2. Enable a **fresh** language with no templates yet (e.g. *French* `fr` — `en`/`es`/`eo`/`de` are
   already generated). It appears in the Supported list as **`[Generating…]`**.
3. **Stay on the Languages tab.** Within one WxReport cycle (≤ ~5 min) the service generates it; the
   row should flip **`[Generating…]` → `[Ready]`** on its own, within ~10 s of the generation
   completing — **without** switching tabs. *(This is the WX-212 behaviour; before it, the row
   stayed `[Generating…]` until you left and re-entered the tab.)*
4. **Selection preserved.** Before it flips, click the generating row — the detail panel shows its
   guidance. Confirm the selection and detail panel **persist** across the 10 s auto-refresh ticks
   (the row is not deselected every tick), and when it reaches READY the detail panel **updates in
   place** to the Ready guidance.
5. **Polling stops when hidden.** Switch to another tab and back — the tab still works normally (and
   the timer only runs while the tab is visible, so there is no background querying while you are
   elsewhere; by design, not directly observable).

Record the outcome in the Jira **Test Result** field as `PASS yyyy-mm-dd @ 1.35.2` (or `FAIL …`).
