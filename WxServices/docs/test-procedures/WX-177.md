# WX-177 Functional Test Procedure — narrative self-consistency, round 2 (post-deployment verification)

**What this verifies:** that WX-177 (≥ 1.47.2) is live and that **live Claude complies** with
the narrative self-consistency rules — the closing/change-band prose no longer contradicts the
snapshot. Two layers: **Defect A** ships a deterministic net (`CheckAggregateDryClaim` — a "the
weekend stays dry" claim contradicted by precip on a weekend day, fail-closed through the WX-189
retry) plus a prompt rule; **Defect B** ships prompt rules only (cite the day's grid-headline
figure; don't over-state a window the grid renders dry). The unit tests prove the validator;
only production proves live Claude complies — the success state is *silence* (clean reports, no
exhausted weekend-dry degrades).

**Recording results:** on pass, set the ticket's custom fields — **Test Proc** =
`docs/test-procedures/WX-177.md`, **Test Result** = `PASS yyyy-mm-dd @ <version>`. On failure:
`FAIL yyyy-mm-dd @ <version> — <which check>`, a ticket comment, and open the Bug (the
`FAIL: Open bug ticket` transition).

## 2. Confirm the rule is live and Claude complies (the PASS/FAIL signal)

Run the script (step 0) after ≥ one full active day past the `1.47.2` boundary.

- **Expect:** *Reports flowed since deploy* > 0 (the fix is live).
- **Expect:** *EXHAUSTED weekend-dry degrades* = **0** — the PASS signal (live Claude scopes
  aggregate periods correctly, or names the specific dry day).
- *Self-correcting weekend-dry retries* ≥ 0 is **fine** — a nudge Claude then obeys is the
  prompt+retry loop working, not a failure.

## 3. If an exhausted weekend-dry degrade fired — adjudicate it

The script prints the flagged lines. Read each:

- **Genuine mis-scope** (the prose really did call the weekend dry over a wet weekend day): live
  Claude isn't complying → **FAIL**; open a Bug to strengthen the Defect A prompt rule (the
  backstop still protected the recipient).
- **False positive** (the flagged prose is legitimate — a correctly-scoped or negated claim the
  check over-matched): → **FAIL**; open a Bug to tighten `AggregateDryWords` / the negation guard.

## 4. WX-283 evidence — did the prompt reduce the category degrades? (informational)

The script prints the *precip-at-a-dry-time* degrades before → after the deploy (equal-length
windows). This does **not** drive the verdict — it is the evidence for **WX-283** (the deferred
deterministic weekday/multi-window resolver):

- **Fell** → the prompt rules are helping; WX-283 likely stays closed as unneeded.
- **Did not fall** → category/weekday-pinned conflicts still recur → this is WX-283's go-signal;
  comment the numbers on WX-283.

## 5. Spot-check a shipped report

Read a recently delivered report that spans a weekend. **Expect:** the closing names the specific
dry day ("Saturday stays dry") rather than calling the whole weekend dry when a weekend day carries
storms, and a cited figure matches the grid row.
