# WX-134 Manual Test Procedure — WxManager locality UX fixes

**What this verifies:** the WX-134 fix set — zombie-preview race, tab-return
reload, locality Delete semantics, click diagnostics, Save/Send
dirty-tracking, unassign-clears-label, and typed-text-is-intent. The WPF apps
have no automated UI harness (and are not CI-built — WX-135), so this
document is the verification record for these behaviors. It also serves as
the standing regression suite for the WxManager locality features
(WX-127 onward).

**Recording results:** when every section passes, set the ticket's custom
fields: **Test Proc** = `docs/test-procedures/WX-134.md` and
**Test Result** = `PASS yyyy-mm-dd @ <version>`. If anything fails, set
`FAIL yyyy-mm-dd @ <version> — section N` and describe the failure in a
ticket comment.

   ```powershell

   dotnet build WxServices.sln
   ```

   Confirm the output ends with **Build succeeded**. (Visual Studio's

3. On the **Setup** tab, wait for the prerequisite checks; all must pass

4. Data prerequisites — check on the **Localities** tab:

5. ⚠️ Sections 3, 6, and 7 change real data. Run against the local dev
   database only, and put assignments back as noted at the end of each
   section.

Throughout: "**Expect:**" lines are pass/fail criteria. If reality differs
from an Expect line, stop, note the section number, and record a FAIL.

## 1. Dirty-tracking — initial state (Recipients tab)

1. Click the **Recipients** tab.
2. Do not click anything else yet. Look at the bottom-right buttons.
   **Expect:** Save, Cancel, and Delete are all **grayed out** (disabled).
   The left-hand list has no highlighted recipient; the form shows defaults
   (Language/Timezone/Scheduled Hours populated; everything else blank).
3. Click into the **Name** box and type `zzz`.
   **Expect:** Save **stays disabled** — you haven't selected a recipient or
   clicked New, so there is nothing to save.
4. Click **New** (bottom of the left pane).
   **Expect:** the form clears to defaults; Save is still **disabled**.
5. Type one character into **Name**.
   **Expect:** Save **enables** on that first keystroke.
6. Click **Cancel**.
   **Expect:** the form clears; Save/Cancel/Delete disabled again.

## 2. Dirty-tracking — the other three tabs

1. **Configure** tab: on arrival, **Save Configuration** is disabled.
   Click into any text box (e.g. SMTP Port) and retype the same value.
   **Expect:** Save Configuration **enables**. Click it.
   **Expect:** status reads "Configuration saved." and the button returns to
   **disabled**. (The app re-runs the Setup checks afterward — that's
   normal; wait for them.)
2. **Recipients** tab: click any recipient in the list.
   **Expect:** fields populate; Save **disabled**. Edit the **Email** (add
   then delete a character is enough).
   **Expect:** Save **enables**. Click Save.
   **Expect:** green "Saved successfully." banner; Save returns to
   **disabled**.
3. **Localities** tab: click a locality in the list.
   **Expect:** fields populate; Save **disabled**. Click into
   **Scheduled Hours**, retype the same value.
   **Expect:** Save **enables**. Click Save.
   **Expect:** green banner mentioning "propagated to N member(s)"; Save
   returns to **disabled**.
4. **Announcement** tab: on arrival, **Send** is disabled. Type a word into
   the big text box.
   **Expect:** Send **enables**. Click **Cancel** (clears the text).
   **Expect:** Send **disables**. (Do NOT click Send — it emails everyone.)

## 3. Tab-return reload + Remove member + label clearing

1. **Recipients** tab: click a locality member (e.g. **bupham-en**).
   **Expect:** METAR ICAO(s), TAF ICAO, Timezone, and Scheduled Hours are
   **read-only** (hover one — tooltip says "Managed by the recipient's
   locality"); the Locality combo shows the locality name.
2. Leave her displayed. Click the **Localities** tab. Select her locality
   (e.g. **Spring, TX**). In the **Members** list, click **Remove** on her
   row.
   **Expect:** she vanishes from the list; the Members count drops by one;
   the Centroid value changes; a green banner says she was removed —
   "stations, timezone, and hours kept; locality label cleared; centroid
   updated."
3. Click the **Recipients** tab. Touch nothing.
   **Expect:** she is still the displayed recipient, but now her four
   mirrored fields are **editable** and the **Locality combo is blank** —
   no leftover locality name anywhere.
4. Restore: go back to **Localities**, select the locality, pick her in the
   **Add member** dropdown, click **Add**.
   **Expect:** she reappears in Members; count and centroid update; green
   banner. (This restores her stations/timezone/hours/label from the
   locality.)

## 4. Locality combo — select, preview, deselect, Save

1. **Recipients** tab: display an **unassigned** recipient (blank Locality
   combo). Note their current METAR ICAO(s) value on paper.
2. Open the Locality combo's dropdown (click the arrow) and click an
   existing locality.
   **Expect:** the four mirrored fields instantly show the **locality's**
   values and become read-only; an amber banner says "Will assign to … on
   Save"; **Save enables**.
3. Click into the combo, select-all (Ctrl+A), press Delete.
   **Expect:** the four fields **revert to the values you noted** and become
   editable again; the amber banner **disappears**.
4. Click **Cancel** to abandon the experiment (nothing was saved).

## 5. Zombie-preview race (repeat 5 times — it's timing-dependent)

1. Display an unassigned recipient. In the Locality combo, slowly type the
   first letter of an existing locality's name (e.g. `S` for "Spring, TX").
   **Expect:** auto-complete may select the locality — fields preview and
   lock, banner appears. That part is normal.
2. Continue typing so the text no longer matches any locality (e.g.
   `Sxxxx`), or backspace everything out.
   **Expect, every repetition:** when the text matches no locality, the
   fields are **unlocked and hold the recipient's own values** — never
   locked, never showing another locality's values. The banner is gone.
3. Click **Cancel** between repetitions to reset cleanly.

## 6. Typed text is intent — the three-way prompt

1. Display an unassigned recipient whose Locality combo shows a stored label
   (e.g. **alex_es** showing `Austin, TX`, assuming no locality of that name
   exists yet).
2. Select-all in the combo and retype the **exact same text**. Click
   **Save**.
   **Expect:** a popup titled "WxManager — Create Locality" with **three**
   choices (Yes / No / Cancel), explaining each.
3. Click **Cancel**.
   **Expect:** nothing saved; amber banner says to adjust the field.
4. Click Save again, choose **No**.
   **Expect:** green banner noting the text was saved "as a display label
   (no locality)". The Localities tab does NOT gain a new entry.
5. Anti-nagware check: with the same recipient, change **only the Email**
   (add/remove a character), do not touch the Locality combo, click Save.
   **Expect:** **no popup** — saves directly, label intact.
6. (Optional, creates real data) Click Save after retyping the label and
   choose **Yes**.
   **Expect:** the locality is created seeded from this recipient (check the
   Localities tab: stations/timezone/hours match theirs; they're its one
   member; centroid equals their coordinates); on the Recipients tab their
   mirrored fields are now locked. Delete it via section 7 if unwanted.

## 7. Locality Delete semantics

1. **Localities** tab → **New**. Name it `Delete Me, TX`, set Timezone
   `America/Chicago`, leave stations blank, **Save**.
2. Add one member via the **Add member** dropdown (anyone unassigned — you
   will restore them after).
3. Click **Delete** (bottom right).
   **Expect:** a confirmation popup that explicitly says its **1 member will
   be unassigned — they keep their stations, timezone, and scheduled hours,
   but their Locality label is cleared**.
4. Click **No**.
   **Expect:** nothing changes — locality and member still present.
5. Click **Delete** again, then **Yes**.
   **Expect:** the locality disappears from the list; the pane goes idle.
   On the **Recipients** tab the ex-member shows editable fields and a
   **blank** Locality combo.
6. Restore the ex-member's intended state (re-add to their real locality,
   or re-enter their label via section 6's "No" path).

## 8. Click diagnostics (log check)

1. Open `C:\HarderWare\Logs\wxmanager.log` in a text editor and go to the
   end.
   **Expect:** every Add/Remove click from sections 3 and 7 left a line
   (e.g. `RemoveMemberBtn_Click fired (locality=…)`), and the
   removals/deletions are narrated. No click you made should be absent.
