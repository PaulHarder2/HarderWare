# WX-258 — editable English source + idiomatic dayparts: manual test procedure

**Change (v1.45.0, WxManager + WxReport.Svc).** The Vocabulary tab's English source column is editable in place (Part A); the stored daypart words are corrected to canonical lower-case through it (Part B); the narrative prompt licenses same-root idiomatic transformation — inflection / compounding / derivation (Part C). The picker marks dormant (disabled) languages "— disabled" (Option C).

**How to run.** Interactive (WORKFLOW §7a.4): the operator is read one step at a time and reports the result; failures are collected and listed at the end. Needs a **local WxManager build of 1.45.0** (WPF is not CI-built) for Parts A/B, and — for Part C — 1.45.0 WxReport.Svc **deployed** plus a daypart-bearing report rendered in **one or two enabled + READY languages that already have a recipient**.

---

## Part A — the English source column is editable (WxManager)

1. Build + launch WxManager (1.45.0) and open the **Vocabulary** tab. *Expect:* it loads; the column headers read **"English"** and **"Selected Language"** (not "English (source)" / "Phrase").
2. Look at the language picker entries. *Expect:* every non-English language with templates is listed; any **disabled** language reads "**\<Name\> (iso) — disabled**"; live ones carry no suffix.
3. Select an enabled language, click into an **English** cell, and change its text (add a word). *Expect:* the cell is editable (not read-only) and "Save changes" enables.
4. In an English cell for a token whose English contains a `{n}` placeholder, delete that placeholder (or blank the cell). *Expect:* the cell flags invalid (red) and **Save is disabled**. Click back into the cell. *Expect:* it reverts to the pre-edit value.
5. Make a valid English-only edit (recase a word — no placeholder change) and Save. *Expect:* "Saved N change(s)"; no error.
6. Switch away to another language and back (or re-select). *Expect:* the English edit persisted.
7. **Per-field stamp check** (I run the DB query at this step). After an English-only edit + Save, the *target* (Selected Language) phrase for that row must be untouched and its review stamp unchanged. *Expect:* target phrase unchanged; only the `en` row was re-stamped.

## Part B — correct the daypart sources to lower-case (WxManager)

8. For each enabled language, find `PartMorning` / `PartAfternoon` / `PartEvening` and lower-case any capitalized daypart in the **English** column (e.g. "Afternoon" → "afternoon") and in the **Selected Language** column (e.g. es "Tarde" → "tarde"). Save. *Expect:* saves cleanly; the daypart words are now lower-case in the grid.

## Part C — idiomatic daypart narrative (deployed WxReport.Svc)

9. With 1.45.0 deployed, trigger a daypart-bearing report (a "rain expected Friday afternoon"-type change) for one or two enabled + READY languages that already have a recipient. *Expect:* the report generates and sends.
10. Read the free narrative (changeSummary / closing) of the produced report. *Expect:* the daypart word is **idiomatic** — not capitalized mid-sentence, rendered in the natural form for a compounding/deriving language (a compounded or adverb form, not the bare dictionary citation word), and it is the approved root, not a synonym.
11. Confirm the deterministic cells (hazard banner / WX-189 fallback band, if present) still render the daypart lower-cased. *Expect:* lower-case, unchanged from before.
