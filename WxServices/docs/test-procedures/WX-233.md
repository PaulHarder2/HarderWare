# WX-233 — WxManager Vocabulary maintenance tab

Manual UI test for the per-language `LanguageTemplates` editor. The shared placeholder guard
(`TemplateValidation.PlaceholdersMatch`) is covered by automated tests
(`WxReport.Tests/TemplateValidationTests.cs`); this procedure covers the WPF editor.

**Build/run:** `dotnet build WxServices.sln`; WxManager exe at
`C:\HarderWare\BuildCache\WxServices\WxManager\bin\Debug\net8.0-windows\WxManager.exe`.

## Steps

1. Launch WxManager; let Setup checks pass; open the **Vocabulary** tab.
   *Expect:* a **Language** selector (target languages only — **no English**), a **Save changes** button
   (disabled), and a grid of that language's tokens.

2. Inspect the grid.
   *Expect:* columns Token, English (source), Phrase, Note, Context, Repr. **Phrase and Note are
   editable**; Token / English / Context / Repr. are read-only.

3. Edit a `Phrase` to a valid value.
   *Expect:* **Save changes** enables ("N change(s) ready to save.").

4. Manually revert that Phrase to its original text (also try a Note edit then revert).
   *Expect:* **Save disables** again ("No changes.") — dirty tracking is value-based.

5. **Empty validator:** clear a Phrase.
   *Expect:* the cell flags red (persistent dark-red highlight, not just while editing) and Save disables.

6. **Placeholder validator:** edit a `{n}`-bearing token (e.g. eo `ForecastHeadingFormat` → drop the `{0}`).
   *Expect:* the cell flags red and Save disables — the guard makes an invalid Phrase **unsaveable**, not
   merely warned. (Editing a placeholder-free token like `GridTimeLegend` is unaffected.)

7. **Tooltip responsiveness:** hover a flagged cell.
   *Expect:* the explanation tooltip appears in ~0.2s (not the WPF ~1s default) and reads the validator
   message ("Phrase cannot be empty." / "Phrase must keep the English source's {n} placeholders …").

8. **Multiple errors:** break a second Phrase before fixing the first.
   *Expect:* both flag red; Save stays disabled until **every** cell is valid; each is independent (no
   modal lock to one cell), and no invalid value can be written.

9. **Restore on re-entry:** click back into a flagged Phrase cell.
   *Expect:* it snaps back to the pre-edit value and the red flag clears.

10. **Save writes + stamps (DB-verified):** edit a placeholder-free token (e.g. eo `AndConjunction` → a test
    value) → **Save changes**.
    *Expect:* status "Saved N…"; in `LanguageTemplates` for that (language, token), `Phrase` is updated,
    `ReviewedBy` = `WX-233 vocabulary editor`, and `ReviewedAtUtc` is stamped (confirm via SSMS). The
    WxReport.Svc template cache picks it up next cycle. (Restore the test value afterward.)
