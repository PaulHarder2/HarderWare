# WX-218 — judge round-trip (manual functional verification)

**Type:** Manual functional verification. The response *parser* is unit-tested (`JudgeResponseParserTests`); this procedure verifies the one thing no unit test can — a **real round-trip** through an independent model (Copilot/ChatGPT) parses cleanly end to end.

**Component:** `WxReport.Tools.TranslationQa` console tool (dev tool; ships in no service binary), run against the live `WeatherData` DB.

## Shell & working directory

Both phases run in **Windows PowerShell** (not WSL — `dotnet` is on the Windows PATH) on the HarderWare PC, from the **solution root**:

```text
C:\Users\PaulH\Dropbox\PH\Documents\Code\HarderWare\WxServices
```

Every command below is relative to that directory (the `--project src\…` path resolves from there), so `cd` to it first. Each step repeats the `cd` so a block can be copy-pasted on its own.

## Preconditions

- A target language that is **enabled + generated (READY)** in the DB (e.g. `de`, `es`, `eo`).
- The tool builds; the DB has `GlobalSettings.ClaudeApiKey` (generate makes one live Claude call per scenario).

## Procedure

1. **Generate** (Phase 1) — Windows PowerShell, from the solution root:

   ```powershell
   cd C:\Users\PaulH\Dropbox\PH\Documents\Code\HarderWare\WxServices
   dotnet run --project src\WxReport.Tools.TranslationQa -- --lang de
   ```

   - **Expect:** four `.html` files + `de.<stamp>.request.md` + `de.<stamp>.request.json` + `de.<stamp>.response.txt` in `C:\HarderWare\translation-qa`.
   - **Expect:** a "Next steps" cue card prints the request path, the response path, and the exact `--response` command.
   - **Expect:** double-clicking `de.<stamp>.response.txt` opens Notepad showing the paste instruction + the re-run command.

2. **Round-trip through the independent model.** Open `de.<stamp>.request.md`, copy all, paste into **Copilot or ChatGPT** (not Claude). Copy its full reply. Open `de.<stamp>.response.txt`, select-all, paste the reply over the placeholder, save.

3. **Judge** (Phase 2) — Windows PowerShell, from the same solution root; run the command the cue card printed (the response path is absolute, but the `--project` path still needs the solution-root `cd`):

   ```powershell
   cd C:\Users\PaulH\Dropbox\PH\Documents\Code\HarderWare\WxServices
   dotnet run --project src\WxReport.Tools.TranslationQa -- --response "C:\HarderWare\translation-qa\de.<stamp>.response.txt"
   ```

   - **Expect:** `Parsed verdict for 'de'.` then a summary (confidence + counts of back-translations / report findings / vocabulary verdicts).
   - **Expect:** `de.<stamp>.judged.json` is written — a clean, indented `JudgeResponse` with native script preserved (ü/ß literal, not `\uXXXX`).
   - **Watch point:** the model's reply is parsed despite any ```json fence or surrounding prose; no crash.

4. **Bad-paste guard.** Save a file with junk (no JSON) and point `--response` at it — Windows PowerShell, same solution root:

   ```powershell
   cd C:\Users\PaulH\Dropbox\PH\Documents\Code\HarderWare\WxServices
   dotnet run --project src\WxReport.Tools.TranslationQa -- --response "C:\HarderWare\translation-qa\junk.response.txt"
   ```

   - **Expect:** a clear `error: couldn't parse the reply — …` with guidance to paste the JSON and re-run; exit code 1; no `judged.json` written.
