# WX-213 — Test Procedure

**Change:** The WxManager Languages tab's status/guidance `DetailPanel` was unreadable in the dark
theme — it hardcoded a light `Background="#FFF7F7F7"` / `BorderBrush="#FFCCCCCC"` while its text
inherits the theme's light foreground (`#c8c8c8`), so the guidance rendered light-on-light. Repainted
for the dark theme: `Background="#252525"`, `BorderBrush="#404040"`.

**Ships in:** WxManager (`LanguagesTab.xaml`), release **1.35.3**. WxManager is **not CI-built**.

**No `WX-213-verify.sh`:** a visual/theme fix with no durable log/DB fingerprint — verified by eye.

---

## 1. Build (pre-merge)

```bash
dotnet build WxServices/WxServices.sln
```

**Expect:** build succeeds, 0 warnings (WxManager included).

## 2. Functional verification (manual, on the deployed WxManager)

1. Launch the deployed **WxManager** (1.35.3) and open the **Languages** tab.
2. Click a **supported** language (e.g. `de` / `fr` / `pt`, all Ready, or enable a fresh one to see
   `[Generating…]`).
3. **Expect:** the guidance panel below now shows **legible text** on a dark panel — a header
   (`Name (iso)  [Ready]/[Generating…]/[Blocked]/[Failed]`) and the guidance paragraph beneath it.
   *(Before WX-213: the panel grew but the text was invisible — light-on-light.)*
4. If convenient, confirm legibility across states: a **Ready** language (e.g. `de`) shows the
   "Ready — generated … recipients can be assigned" line; a freshly-enabled one shows the
   "Generating — queued …" guidance.

This also closes the visual loop on **WX-212**: with the panel readable, you can confirm the
auto-refresh updates the guidance **in place** (e.g. `[Generating…] → [Ready]`) without flicker and
without losing the selected row.

Record the outcome in the Jira **Test Result** field as `PASS yyyy-mm-dd @ 1.35.3` (or `FAIL …`).
