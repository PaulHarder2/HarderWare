# WX-249 — non-destructive language disable: post-deploy verification

**Change (v1.44.0, WxReportSvc + WxManager).** Disabling a report language used to DELETE all of its `LanguageTemplates` rows and revert it to needs-generation (WX-222). Under the WX-247 en-only model those rows are generated + human-QA-curated (`ReviewedBy`/`ReviewedAtUtc`), so the purge is a silent data-loss hazard. New invariant: **curated templates are durable data; `IsEnabled` gates _use_, not _existence_.** Disable flips `IsEnabled=false` and KEEPS the rows + the `GeneratedAtUtc` stamp; the WX-171 startup completeness check is scoped to **enabled** languages (a dormant-incomplete language no longer trips a false `INCOMPLETE` ERROR); the generator no longer discards a top-up for a language disabled mid-call. Re-enable reuses the rows; WX-250 fills only baseline gaps opened while disabled.

**Boundary.** The 1.44.0 `WxReportSvc` deploy — `deploy-info.sh --version 1.44.0 WxReportSvc`.

**Gate (agreed with Paul, 2026-07-04).** Two parts: (1) the deployed service's scoped completeness check runs clean (automated); (2) an operator disable→re-enable of a reviewed language preserves its rows + review stamps (manual DB check — the core AC). The rigorous preservation proof is the `LanguageToggleTests` unit suite; the manual check confirms it end-to-end in the deployed WxManager + real DB.

---

## 1. Automated check (deploy health + scoping live)

After the 1.44.0 deploy, run:

```
bash docs/test-procedures/WX-249-verify.sh
```

Over the `WxReport.Svc` log since the version-pinned boundary it reports:

- **Completeness-check regression** — any `is INCOMPLETE` ERROR (an enabled language missing a hard token, or the scoping leaking a disabled language). *Expect:* **none**.
- **Scoped completeness ran** — the `completeness check passed for all <N> enabled language(s)` startup line; the word **enabled** (old code said "loaded") proves the 1.44.0 scoping is live. *Expect:* ≥1.
- **Background health** — new `ERROR` lines vs the equal-length pre-deploy window.

**Verdict:** `PASS` past the 5-min window with cycles run, the scoped line present, and no INCOMPLETE; `WAIT` before the deploy/first cycle; `FAIL` on an INCOMPLETE regression. Record the printed `Test Result` string in the Jira **Test Result** field.

## 2. Manual check — the core AC (disable is non-destructive)

Pick an **enabled, generated** target language carrying a human-reviewed row — e.g. Albanian (`sq`, `VisHazy → Turbullt`) or Spanish (`es`, `PartEvening → Tarde-Noche`).

**2a. Capture the reviewed rows before disable (PowerShell):**

```
sqlcmd -S .\SQLEXPRESS -d WeatherData -E -C -Q "SET NOCOUNT ON; SELECT t.Token, t.Phrase, t.ReviewedBy, t.ReviewedAtUtc FROM LanguageTemplates t JOIN Languages l ON l.Id=t.LanguageId WHERE l.IsoCode='sq' AND t.ReviewedBy IS NOT NULL ORDER BY t.Token;"
```

**2b. Disable it** in WxManager's Languages tab. *Expect:* status reads "…its curated report templates are preserved; re-enabling reuses them…" (not the old "were cleared").

**2c. Confirm the rows SURVIVED (the payoff).** Re-run the 2a query — *expect **identical rows*** (same `Phrase`, `ReviewedBy`, `ReviewedAtUtc`; before WX-249 they would be gone). Then confirm the language kept its stamp and row count:

```
sqlcmd -S .\SQLEXPRESS -d WeatherData -E -C -Q "SET NOCOUNT ON; SELECT IsoCode, IsEnabled, GeneratedAtUtc, (SELECT COUNT(*) FROM LanguageTemplates t WHERE t.LanguageId=l.Id) AS Rows FROM Languages l WHERE IsoCode='sq';"
```

*Expect:* `IsEnabled=0`, `GeneratedAtUtc` still set, `Rows` unchanged from before the disable.

**2d. No false INCOMPLETE for the disabled language.** With `sq` disabled, restart WxReport.Svc (the completeness check runs at startup). *Expect:* `sq` is **absent** from any `INCOMPLETE` ERROR and is not counted in the "passed for all N enabled language(s)" line — it is dormant, not alerted. (Re-run `WX-249-verify.sh`: still `PASS`.)

**2e. Re-enable it.** *Expect:* status reads "…ready; recipients can now be assigned it." — immediate reuse, no full regeneration — with the reviewed rows untouched. Only baseline tokens added while disabled (e.g. the WX-256 noon/midnight soft tokens, if `sq` lacked them) top up over the next cycles.
