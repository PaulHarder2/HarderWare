# WX-248 — WeatherData automated backups (manual functional verification)

**Type:** Manual functional verification. Local Windows infra (a scheduled task + PowerShell on
the HarderWare box); ships in no service binary, no version bump. The **restore drill (step 5) is
the Done gate** — a backup never restored is not a backup.

**Shell & account:** Windows PowerShell on the HarderWare box. `sqlcmd` uses `-E -C`.
Registration (step 2) must run **elevated**.

## Preconditions

- SQL Server Express (`.\SQLEXPRESS`) up; `WeatherData` present.
- `tools/Backup-Config.json` points `LocalStagingDir` and at least one `Destinations` entry at
  real paths (default: staging `C:\HarderWare\backups`, Dropbox `C:\Users\PaulH\Dropbox\PH\WxBackups`).

## Procedure

1. **Ad-hoc backup run** (from the repo `tools` dir):
   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\Backup-WeatherData.ps1 -Type Full
   ```
   - *Expect:* exit 0; `C:\HarderWare\Logs\weatherdata-backup.log` shows `Backup verified: … SHA256 …`,
     `copied + verified -> …Dropbox\WxBackups\WeatherData-full-<stamp>.bak`, and `Backup OK`.
   - *Expect:* the `.bak` exists in **both** `C:\HarderWare\backups` and the Dropbox destination,
     identical size.
   - *Watch point (server-side write):* if it logs `Access is denied` on the `.bak` path, grant the
     SQL service account write on the staging dir (command in `Register-BackupTask.ps1`), re-run.

2. **Register the scheduled task** (elevated):
   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File .\Register-BackupTask.ps1
   ```
   - *Expect:* `Registered task: HarderWare WeatherData Backup (daily 08:00 UTC)`; script + config
     copied to `C:\HarderWare\backup\`.

3. **Fire the task once** and confirm it runs the installed copy:
   ```powershell
   Start-ScheduledTask -TaskName 'HarderWare WeatherData Backup'
   ```
   - *Expect:* a fresh `Backup OK` line in the log within ~1 min; a new `.bak` in both locations.

4. **Verify a backup file** is internally valid:
   ```powershell
   sqlcmd -S .\SQLEXPRESS -E -C -b -Q "RESTORE VERIFYONLY FROM DISK = N'C:\HarderWare\backups\<newest>.bak' WITH CHECKSUM;"
   ```
   - *Expect:* `The backup set on file 1 is valid.` exit 0.

5. **Restore drill — the Done gate** (non-destructive; restores to a scratch DB, never touches
   `WeatherData`). Per `docs/restore-weatherdata.md` §3a:
   ```powershell
   sqlcmd -S .\SQLEXPRESS -E -C -b -Q "RESTORE DATABASE [WeatherData_restoretest] FROM DISK = N'C:\HarderWare\backups\<newest>.bak' WITH MOVE 'WeatherData' TO N'C:\HarderWare\backups\rt.mdf', MOVE 'WeatherData_log' TO N'C:\HarderWare\backups\rt_log.ldf', RECOVERY, REPLACE;"
   sqlcmd -S .\SQLEXPRESS -E -C -Q "SELECT COUNT(*) FROM WeatherData_restoretest.dbo.LanguageTemplates;"
   sqlcmd -S .\SQLEXPRESS -E -C -Q "DROP DATABASE [WeatherData_restoretest];"
   ```
   - *Expect:* restore completes; the `LanguageTemplates` count matches live `WeatherData`
     (the curated vocabulary came back); scratch DB dropped cleanly.

6. **Retention** (spot check): confirm the prune logic keeps `RetentionDays` and would remove older
   `WeatherData-*.bak/.dif`. (No wait needed — inspect the prune log line / reason about `LastWriteTime`.)

7. **Failure surfaces** (no silent failure): temporarily point a destination `Target` at an
   unwritable path (or a bad config) and run step 1 → *Expect:* an `ERROR` line, a
   `weatherdata-backup.FAILED` sentinel, and non-zero exit. Restore the config after.
