# WX-222 ‚Äî Orphaned LanguageTemplates lifecycle (disable purges; re-enable regenerates)

**Type:** Manual functional verification (post-deploy). Service-log + DB inspection, plus an optional WxManager disable/re-enable cycle.

**Release:** 1.37.1
**Components:** WxManager (`LanguagesTab.SetEnabledAsync`), MetarParser.Data (`PurgeDisabledLanguageTemplates` migration), WxReport.Svc (`ReportWorker` disable-mid-generation guard + the startup completeness check whose fr/pt symptom clears).

## Preconditions

- 1.37.1 deployed; the `PurgeDisabledLanguageTemplates` migration applied at startup.

## Procedure

1. **Startup log of the post-deploy WxReport restart.**
   - **Expect:** no `Language 'fr' is INCOMPLETE ‚Ä¶` / `Language 'pt' is INCOMPLETE ‚Ä¶` ERROR lines.
   - **Expect:** the load line now reports the **five enabled** languages, e.g. `LanguageTemplateStore loaded ~505 template(s) across 5 language(s)` (was 707 across 7), and `Language template completeness check passed for all 5 loaded language(s).`

2. **DB ‚Äî no templates for disabled languages.**

   ```sql
   SELECT l.IsoCode, COUNT(*) AS Tokens
   FROM LanguageTemplates lt JOIN Languages l ON l.Id = lt.LanguageId
   WHERE l.IsEnabled = 0 GROUP BY l.IsoCode;
   ```

   - **Expect:** zero rows (the fr/pt orphans purged). `SELECT IsoCode, GeneratedAtUtc FROM Languages WHERE IsoCode IN ('fr','pt')` ‚Üí `GeneratedAtUtc` is NULL.

3. **WxManager disable ‚Üí re-enable cycle (the invariant).** Pick a throwaway language not assigned to any recipient (e.g. enable `fr`, let it generate to Ready on the next report cycle).
   - **Disable** it on the Languages tab. **Expect:** status reads "‚Ä¶its report templates were cleared; re-enabling will regenerate them"; its `LanguageTemplates` rows are gone and `GeneratedAtUtc` is NULL.
   - **Re-enable** it. **Expect:** it requeues for generation and, on the next report cycle, regenerates a **complete** set (the completeness check passes for it) ‚Äî including any tokens added since it was first generated.
   - **Worker disable-mid-generation guard (narrow timing race, optional):** if you **disable** the language while its `WX-172: generating templates for '‚Ä¶'` log line is active, **expect** no templates to be written for it afterward ‚Äî the worker re-reads `IsEnabled` before persisting and discards the result (log: `was disabled during generation ‚Äî discarding the result`). Hard to time by hand; documented so the behavior is on record.

4. **Deletion capability (optional, non-destructive check).** A disabled language now has no templates, so the `LanguageTemplate ‚Üí Language` FK no longer blocks deleting the `Language` row. Verify only (do not actually delete unless intended). There is no delete UI ‚Äî capability only (see ticket non-goals).
