# WX-66 Manual Test Procedure — Containerized WxParser.Svc

**What this verifies:** `WxParser.Svc` runs correctly as a Linux Docker container
(`services/docker-compose.yml`, service `wxparser`) instead of a Windows service — it builds,
reaches the host SQL Server, starts, fetches METAR/TAF (HTTP) and GFS GRIB2 (byte-range) and writes
them to the DB, and — the crux — its **bundled Linux `wgrib2`** decodes GFS GRIB2 and extracts the
sub-grid (`-small_grib` → `-csv`) exactly as the host's `wgrib2.exe` does. Landing this completes the
WX-7 arc: all four services are containers, and `Deploy-WxService.ps1` is fully container-based.

**Recording results:** when every section passes, set the ticket's custom fields:
**Test Proc** = `docs/test-procedures/WX-66.md` and **Test Result** = `PASS yyyy-mm-dd @ <version>`.
If anything fails, set `FAIL yyyy-mm-dd @ <version> — section N` and describe it in a ticket comment.

"**Expect:**" lines are pass/fail criteria — if reality differs, note the section and continue.

---

## 0. Setup / precondition

```powershell
$RepoRoot    = 'C:\Code\HarderWare'
$InstallRoot = (Get-Content "$RepoRoot\WxServices\appsettings.shared.json" -Raw | ConvertFrom-Json).InstallRoot
# Point the compose Logs bind-mount at the resolved host InstallRoot (forward slashes for Docker),
# as Deploy-WxService.ps1 does; without this a non-default InstallRoot misplaces the log/heartbeat.
$env:WX_HOST_LOGS_DIR = (Join-Path $InstallRoot 'Logs') -replace '\\', '/'

docker info | Select-String 'Server Version'
Test-Path "$RepoRoot\services\wxparser\appsettings.local.json"
```

**Expect:** a server version prints; the path test returns `True`. (Copy
`appsettings.local.json.example` → `appsettings.local.json`, fill the SQL password from RoboForm;
the container overrides carry the DB connection string, the OTel endpoint, and
`Gfs:Wgrib2Path` → `/usr/local/bin/wgrib2`.) **Because WxParser writes to the DB, stop + disable the
native `WxParserSvc` first** (from an elevated shell) so the two don't dual-fetch/dual-write:

```powershell
# elevated
Stop-Service WxParserSvc ; Set-Service WxParserSvc -StartupType Disabled ; Get-Service WxParserSvc
```

## 1. Build + start the container

```powershell
cd "$RepoRoot\services"
docker compose up -d --build wxparser
$cid = docker compose ps -q wxparser   # this run's container id - scope log checks to it
docker compose ps
```

**Expect:** the image builds (COPYs the Linux `wgrib2` + apt `libgfortran5`/`libgomp1`) and the
container starts; `docker compose ps` shows `wxparser` as `running`/`Up`.

## 2. Startup banner + prerequisites (DB reached, wgrib2 runs in-image)

```powershell
docker logs $cid | Select-String 'starting\.','Database ready','Prerequisite OK','Application started'
```

**Expect:** `WxParser.Svc <version> starting.`, `Database ready.` (`EnsureSchemaAsync` reached
`host.docker.internal,1433` via SQL auth), `Prerequisite OK: SQL Server …`, and
**`Prerequisite OK: wgrib2 — wgrib2 v3.1.3 …`** (the service ran the bundled Linux binary), then
`Application started.`.

## 3. METAR/TAF fetch + heartbeat through the mount (the HTTP + InstallRoot proof)

```powershell
Get-Content "$InstallRoot\Logs\wxparser-svc.log" -Tail 30 | Select-String 'METAR fetch done','TAF fetch done'
(Get-Item "$InstallRoot\Logs\wxparser-heartbeat.txt").LastWriteTimeUtc
```

**Expect:** `METAR fetch done. Inserted: N …` / `TAF fetch done. Inserted: N …` lines (the HTTP fetch
+ DB write work), and `wxparser-heartbeat.txt` holds a **recent** UTC timestamp — its landing in the
mounted `Logs` (not a `C:\HarderWare` path inside the container) proves the `AddInstallRoot` seam
resolved to `/opt/wxservices`.

## 4. GFS pipeline — the containerized `wgrib2` decodes real GRIB2 (the crux)

A live GFS fetch inserts new `GfsGrid` rows only when hours aren't already stored, so to prove the
extraction deterministically, feed the container's `wgrib2` a live GFS subset directly — the exact
two calls `GribExtractor` makes:

```powershell
$run = (Get-Date).ToUniversalTime().ToString('yyyyMMdd')   # today's date; pick a run (12) that's published
curl.exe -sS -o "$env:TEMP\gfstest.grb2" "https://nomads.ncep.noaa.gov/cgi-bin/filter_gfs_0p25.pl?dir=%2Fgfs.$run%2F12%2Fatmos&file=gfs.t12z.pgrb2.0p25.f000&var_TMP=on&lev_2_m_above_ground=on&leftlon=254&rightlon=272&toplat=38&bottomlat=25&subregion=on"
docker cp "$env:TEMP\gfstest.grb2" "$($cid):/tmp/gfstest.grb2"
docker exec $cid /usr/local/bin/wgrib2 /tmp/gfstest.grb2 -small_grib 254:272 25:38 /tmp/sub.grb2
docker exec $cid /usr/local/bin/wgrib2 /tmp/sub.grb2 -csv /tmp/out.csv
docker exec $cid head -3 /tmp/out.csv
```

**Expect:** the CSV rows carry sane values, e.g.
`"…","…","TMP","2 m above ground",-106,25,285.3` (≈285 K = 12 °C) — proving the container's `wgrib2`
decodes GFS GRIB2, crops it, and emits the CSV `GribExtractor.ParseCsv` reads. (Equivalently, over
time watch `wxparser-svc.log` for a `GfsFetcher: … N records inserted this cycle` with N > 0 — a
natural extraction — when a new GFS run publishes.)

## 5. Telemetry to the collector / Grafana

**Expect:** `wxparser_fetch_cycles_total` / `wxparser_gfs_cycles_total` (and friends) are present —
the `wxparser` container reaches the collector at `host.docker.internal:4318`.

## 6. Full deploy + status summary (all four now containers)

```powershell
# elevated - the deploy script lives in WxServices\, not services\
cd C:\Code\HarderWare\WxServices
.\Deploy-WxService.ps1 WxParserSvc   # single service first
.\Deploy-WxService.ps1 all           # then the whole stack
```

**Expect:** `WxParserSvc` alone reports its container reached `Application started`; `all` ends with
the summary showing `[OK]` for all six components, now **all four as containers**
(WxParserSvc/WxVisSvc/WxReportSvc/WxMonitorSvc `Application started`; WxManager/WxViewer `published`).
The cutover leaves each native service stopped + disabled.

## Cleanup / cutover note

If the run **passes**, the container stays up as WxParser (the native `WxParserSvc` remains disabled;
`sc delete` all four native services once the containers are proven). If the run **fails**, take the
container down and investigate:

```powershell
cd C:\Code\HarderWare\services; docker compose rm -sf wxparser   # 'down' takes no service name
```
