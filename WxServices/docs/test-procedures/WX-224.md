# WX-224 — Phase B: localized in-image meteogram labels (per-language render)

**Type:** Manual functional verification (post-deploy). WxVis service-log + DB inspection + visual inspection of rendered meteograms in real reports.

**Release:** 1.38.0 (in-image labels) + 1.38.1 (caption-legend readability follow-on).
**Components:** `meteogram.py`, `MeteogramWorker` (WxVis.Svc), `ReportWorker.FindMeteogramAbbrevPath` (WxReport.Svc), migration `AddMeteogramLabelTokens`. 1.38.1 adds `ReportWorker.InsertMeteogramImage` (WxReport.Svc — caption styling only).

## Preconditions

- 1.38.1 deployed (carries the 1.38.0 in-image label work plus the 1.38.1 caption-readability follow-on); `AddMeteogramLabelTokens` applied at startup.
- At least one **non-English** recipient (e.g. a `*_es` recipient) and one English recipient assigned to a locality, so both languages are in demand.

## Procedure

1. **DB — the label tokens seeded.**

   ```sql
   SELECT l.IsoCode, lt.Token, lt.Phrase
   FROM LanguageTemplates lt JOIN Languages l ON l.Id = lt.LanguageId
   WHERE lt.Token LIKE 'meteogram[_]%'
   ORDER BY l.IsoCode, lt.Token;
   ```
   - **Expect:** `meteogram_wind`/`_rh`/`_temp` for en/es/de/da/eo — e.g. es = Viento/HR/T, de = Wind/RF/T, en = Wind/RH/T. The `MeteogramCaption` now reads "…temperature (T), relative humidity (RH/HR)…".

2. **WxVis MeteogramWorker log, on the first complete GFS run after deploy.**
   - **Expect:** `rendering meteograms for run … — N location/language combo(s)` then per-combo `rendering KXXX (…) [es] …` / `[en] …` lines. **Watch point:** no EF/SQL exception (confirms the per-language token query translates).
   - **Expect:** the plots dir gains per-language PNGs, e.g. `meteogram_…_KXXX_…_F_es_abbrev.png` and `…_en_abbrev.png`. Only languages with recipients at that location appear (no blind 5×).

3. **Visual — the core check.** Await/trigger a report to the **es** recipient and the **en** recipient.
   - **es meteogram — Expect:** Wind→**Viento**, RH→**HR**, weekday ticks **localized to Spanish (from `CultureInfo`), Monday-first** — Spanish abbreviations (e.g. lun/mar/mié…, uppercased on the chart), not English Mon/Tue. **Watch point:** the accented abbreviations render correctly — not mojibake — confirming the labels survive the command line to Python.
   - **en meteogram — Expect:** unchanged — Wind/RH/T, MON/TUE/… .

4. **Caption legend — content.** Beneath each meteogram, the HTML caption now **defines** the abbreviations in the recipient's language ("…temperature (T), relative humidity (HR)…" for es). Confirm the caption abbreviation matches the on-chart label (HR == HR).

5. **Caption legend — readability (1.38.1).** Inspect the caption beneath the meteogram in any report (any language).
   - **Expect:** the caption renders at the **same size and color as the "Reported Conditions" station-and-time note and the forecast-grid "Hours" legend** — 13px, muted blue-gray (`#6b8fa8`), italic — not the former smaller, fainter gray. It remains **centered** beneath the image (the in-flow captions are left-justified; the meteogram caption is deliberately centered). All three small-print captions read as one consistent style.
