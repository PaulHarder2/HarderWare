# WX-224 ‚Äî Phase B: localized in-image meteogram labels (per-language render)

**Type:** Manual functional verification (post-deploy). WxVis service-log + DB inspection + visual inspection of rendered meteograms in real reports.

**Release:** 1.38.0
**Components:** `meteogram.py`, `MeteogramWorker` (WxVis.Svc), `ReportWorker.FindMeteogramAbbrevPath` (WxReport.Svc), migration `AddMeteogramLabelTokens`.

## Preconditions

- 1.38.0 deployed; `AddMeteogramLabelTokens` applied at startup.
- At least one **non-English** recipient (e.g. a `*_es` recipient) and one English recipient assigned to a locality, so both languages are in demand.

## Procedure

1. **DB ‚Äî the label tokens seeded.**

   ```sql
   SELECT l.IsoCode, lt.Token, lt.Phrase
   FROM LanguageTemplates lt JOIN Languages l ON l.Id = lt.LanguageId
   WHERE lt.Token LIKE 'meteogram[_]%'
   ORDER BY l.IsoCode, lt.Token;
   ```
   - **Expect:** `meteogram_wind`/`_rh`/`_temp` for en/es/de/da/eo ‚Äî e.g. es = Viento/HR/T, de = Wind/RF/T, en = Wind/RH/T. The `MeteogramCaption` now reads "‚Ä¶temperature (T), relative humidity (RH/HR)‚Ä¶".

2. **WxVis MeteogramWorker log, on the first complete GFS run after deploy.**
   - **Expect:** `rendering meteograms for run ‚Ä¶ ‚Äî N location/language combo(s)` then per-combo `rendering KXXX (‚Ä¶) [es] ‚Ä¶` / `[en] ‚Ä¶` lines. **Watch point:** no EF/SQL exception (confirms the per-language token query translates).
   - **Expect:** the plots dir gains per-language PNGs, e.g. `meteogram_‚Ä¶_KXXX_‚Ä¶_F_es_abbrev.png` and `‚Ä¶_en_abbrev.png`. Only languages with recipients at that location appear (no blind 5√ó).

3. **Visual ‚Äî the core check.** Await/trigger a report to the **es** recipient and the **en** recipient.
   - **es meteogram ‚Äî Expect:** Wind‚Üí**Viento**, RH‚Üí**HR**, weekday ticks **localized to Spanish (from `CultureInfo`), Monday-first** ‚Äî Spanish abbreviations (e.g. lun/mar/mi√©‚Ä¶, uppercased on the chart), not English Mon/Tue. **Watch point:** the accented abbreviations render correctly ‚Äî not mojibake ‚Äî confirming the labels survive the command line to Python.
   - **en meteogram ‚Äî Expect:** unchanged ‚Äî Wind/RH/T, MON/TUE/‚Ä¶ .

4. **Caption legend.** Beneath each meteogram, the HTML caption now **defines** the abbreviations in the recipient's language ("‚Ä¶temperature (T), relative humidity (HR)‚Ä¶" for es). Confirm the caption abbreviation matches the on-chart label (HR == HR).
