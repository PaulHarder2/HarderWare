# WX-171 — renderer rewire no-regression procedure + intended-delta inventory

The renderer rewire (Step 5: read atomic tokens/format strings from the DB, delete the
vocabulary-gluing logic) is gated by a **golden-master** harness. This document is the
procedure and the **allow-list of intended output changes** the gate is checked against.

## The harness

- `tests/WxReport.Tests/Golden/RendererGoldenCorpus.cs` — a deterministic matrix of
  renderer inputs (38 scenarios × en/es + one unit-coverage case = **93 cases**) whose
  union exercises every renderer-reachable `LanguageTemplates` token in both languages.
- `tests/WxReport.Tests/Golden/RendererGoldenTests.cs`
  - `Verify` (`[Theory]`) — renders each case and asserts byte-equality against its
    committed fixture in `Golden/Fixtures/`. Runs in CI.
  - `Record` (`[Fact]`) — rewrites the fixtures; **gated** on `WX171_RECORD=1` so CI never
    rewrites the goldens it is meant to check.

### Capturing / refreshing goldens

```
WX171_RECORD=1 dotnet test tests/WxReport.Tests/WxReport.Tests.csproj \
  --filter FullyQualifiedName~RendererGoldenTests.Record
```

The fixtures committed on this branch were recorded from the **pre-rewire** renderer
(hard-coded `ReportVocabulary`). The no-regression proof: after the rewire, re-record and
`git diff` the fixtures — the diff **must equal the inventory below**. Any other changed
line is an unintended regression to fix; every line below must appear (the rewire is
incomplete if an intended delta is missing).

## Coverage notes

- Every token reachable through `StructuredReportRenderer` is exercised by ≥1 case
  (confirmed by cross-checking each token's rendered phrase against the golden union).
- **Subject-line tokens** (`ScheduledReportSubject`, `UnscheduledUpdateSubject`,
  `DiagnosticSubject`, `SubjectForConnective`, `WelcomeSubject`) are built in
  `ReportWorker`, not the renderer — covered separately when that call site is rewired.
- **Currently unreachable tokens:** `sev_wx_possible`, `sev_wx_expected`. The renderer
  hard-codes "likely" for a non-convective severe day (`OutlookLikely`), so only
  `sev_wx_likely` is emitted. The other two are seeded for completeness and stay dormant
  unless the renderer is later generalized (out of WX-171 scope).

## Implementation action item (must produce NO delta)

The atomic tokens `sky_overcast_low/high`, `sky_mostlycloudy_low/high`, and `clear_and_dry`
are **lowercase** in the seed ("low overcast", "clear and dry"), but they render in cells
the current renderer shows **capitalized** ("Low overcast", "Clear and dry"). The rewired
renderer **must capitalize** the sky / weather / conditions cell (as the current code does
via the prefix word and `SkyWord`/`Capitalize`) so English output is unchanged. If the cell
is filled by raw substitution, English regresses to lowercase — which would show up as an
**un-allowed** delta in the golden diff (the gate catches it). With capitalization handled,
these tokens produce no English delta; the Spanish order change above still applies.
