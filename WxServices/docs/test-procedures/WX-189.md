# WX-189 — Functional test procedure

**Ticket:** WX-189 — Deterministic change detection: compute the "What's changed" set in
code; leave Claude only to select + narrate (eliminate phantom-change invention at the
source).
**Change under test:** the "What's changed" set is no longer authored by the LLM. Claude
returns only `final_snapshot` + narrative prose; `DeterministicChangeDetector` computes the
`changes[]` from (prior committed snapshot, reconciled `final_snapshot`) after the call and
the reconciler injects it. A structural phantom is therefore impossible by construction.

1. **Deterministic detector** — `DeterministicChangeDetector.Detect` is the generative
   inverse of `ForecastReconciler.ValidateChangeSnapshotConsistency`: precip + standalone
   severe by inverting the oracle; temperature + wind via the WX-160 significance-gate
   thresholds (freeze/thaw, heat-advisory, wind-advisory, per-tier magnitude). Obscuration
   is excluded (never populated); WindShift is the sole residual → WX-191.
2. **Tool + validators rewired** — `changes[]` removed from the `submit_reconciled_report`
   tool schema; `{chN}` anchoring retired (`StructuredReportBody.Validate` forbids anchors
   in both prose sections); `ValidateAnchoredProseTiming` deleted; the consistency
   validator now runs as tautological defense-in-depth on the computed set.
3. **Graceful degradation + prose-retry pin** — the change band gains a deterministic
   fallback (rendered from the computed changes when Claude's prose is rejected/absent); a
   prose-fault retry (`NarrativeProseException`) pins `final_snapshot` and re-authors only
   the prose; the precip-vs-dry guard now covers the `changeSummary` too.
4. **Temperature payoff** — `ReconcilerTemperature` raised 0.25 → 0.5; structural
   correctness no longer rides on sampling, so the prose breathes.

**Released in:** 1.32.0 · **Component:** WxReport.Svc

## Deterministic proof (the primary verification)

Unit tests (run with `dotnet test WxServices/WxServices.sln`):

- `tests/WxReport.Tests/DeterministicChangeDetectorTests.cs` — the detector per
  phenomenon × direction (precip, severe de-dup, temperature freeze/thaw/heat/magnitude,
  wind advisory/magnitude), windowing, salience ranking + token assignment, horizon
  bounds, first-send, and **the keystone invariant**: every change the detector emits
  passes `ValidateChangeSnapshotConsistency` (the defense-in-depth net stays tautologically
  green).
- `tests/WxReport.Tests/ChangeConsistencyValidatorTests.cs` — the validator's reject paths
  preserved as direct unit tests (phantom appearing/weakening, off-grid window,
  safety-not-backed).
- `tests/MetarParser.Tests/StructuredReportBodyTests.cs` — anchors now rejected in both
  prose sections; the old anchor↔change correspondence retired.
- `ForecastReconcilerTests.ReconcilerRequest_PinsSamplingTemperatureLow` — the request body
  carries `"temperature":0.5`.
- `ForecastReconcilerTests.ProseRejection_RetryFeedback_PinsSnapshotAndReauthorsProse` — a
  prose fault replays the snapshot-pinning feedback ("Keep your final_snapshot EXACTLY",
  "Re-author ONLY the narrative prose").

## Post-deploy verification (§13)

Script: `docs/test-procedures/WX-189-verify.sh` (version-pinned to 1.32.0 via
`deploy-info.sh`; 24h window). Two tiers:

### Gated — no structural change-consistency degrade

A degrade whose reason is a change-consistency message was, before WX-189, the dominant
degrade cause (a Claude-authored phantom the validator rejected). After WX-189 it can only
arise if the deterministic detector emits a change its own inverse validator rejects — a
detector bug. The script gates on:

- **PRECONDITION / EXERCISED:** a `sending startup (diagnostic) report` line in the window
  (a deploy logs exactly one, so the new detection path ran).
- **REGRESSION (fail) signature:** a `degrading to the parsed snapshot …` line whose reason
  matches `is not a real change versus prior_snapshot` / `is not aligned to a snapshot block
  boundary` / `carries a safety-grade signal`. Expect **0**.

```bash
bash docs/test-procedures/WX-189-verify.sh
```

*Expect:* once a startup diagnostic has run and 24h of cycles have accrued with no
structural change-consistency degrade → **PASS**. Such a degrade → **FAIL** at any horizon.
No startup diagnostic in the window yet → **WAIT**.

### Reported — the general degrade rate (human judgment)

With the structural cause eliminated, the *general* reconciler degrade rate should drop
materially. The residual is prose-fault degrades only. The script **reports** (does not gate
on) the pre/post degrade counts and per-day rate:

- *Expect:* the post-deploy reconciler degrade rate is **materially below** the pre-fix
  baseline of **~16.5/day** (the ~33 events / 2 days observed before WX-165). A flat or
  higher rate means prose faults now dominate the residual — investigate even when the
  structural gate passes.

### Calibration spot-check (manual)

Pull a handful of live reports for a locality with active weather and confirm the
deterministic detector surfaces the changes a forecaster would expect — not over- or
under-producing versus the prior LLM behavior — and that any fallback band reads correctly
in en and es. This is the "calibration risk" the groom flagged; it is a judgment only the
forecaster can make against real forecasts.

Cross-check the §13 baseline smoke (services up, heartbeat current, 1.32.0 running, no new
ERRORs above the equal-length pre-deploy window) — the verify script's BACKGROUND HEALTH
section reports the ERROR delta.
