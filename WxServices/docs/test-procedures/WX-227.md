# WX-227 — automated Gemini judge (manual functional verification)

**Type:** Manual functional verification (live Gemini API). The `GeminiJudge` envelope-extraction + error handling are unit-tested (`GeminiJudgeTests`); this confirms the one thing no unit test can — a **real Gemini API round-trip** ingests the full request and returns a complete, parseable verdict in one command.

**Component:** `WxReport.Tools.TranslationQa` console tool (dev tool; ships in no service binary). No version bump. (WX-227 originally kept the key in the tool-local `appsettings.local.json`; **WX-235 later moved it into the DB** — `GlobalSettings.GeminiApiKey`, set via WxManager → Configure — which is where the tool now reads it. Updated 2026-07-02 during WX-220.)

## Shell & working directory

Run in **Windows PowerShell** (not WSL — `dotnet` is on the Windows PATH) on the HarderWare PC, from the **solution root**:

```text
C:\Code\HarderWare\WxServices
```

## Preconditions

- A Gemini API key (Google AI Studio, free tier) set **in the database** via **WxManager → Configure** — the tool reads it from `GlobalSettings.GeminiApiKey` (WX-235), never from a config file. An optional `Gemini` section in the InstallRoot overlay `C:\HarderWare\appsettings.local.json` supplies only non-secret overrides (model, request timeout); its `ApiKey`, if present, is ignored (always overwritten by the DB value):

  ```json
  { "Gemini": { "Model": "gemini-2.5-flash" } }
  ```

  `Model` is optional (defaults to a flash model). The key is never committed, logged, or stored in a config file.
- A target language **enabled + generated (READY)** in the DB (e.g. `de`).
- DB reachable and `GlobalSettings.ClaudeApiKey` set (the generate phase still makes the Claude reconciliation calls).

## Procedure

1. **One-command generate + judge** — Windows PowerShell, from the solution root:

   ```powershell
   cd C:\Code\HarderWare\WxServices
   dotnet run --project src\WxReport.Tools.TranslationQa -- --lang de --judge gemini
   ```

   - **Expect:** the en+de reports render, then `Judging (en + de) via the Gemini API (…) …`, then `✓ judged by Gemini` with a summary (confidence + counts of back-translations / report findings / vocabulary verdicts) and `→ verdict …judged.json`.
   - **Watch point:** the **full vocabulary** is judged (no chunking, no paste) — vocabulary-verdict count reflects the audited token set, not a hand-fed subset.

2. **Inspect** `de.<stamp>.judged.json` — a complete `JudgeResponse`: language, confidence, both back-translations, report findings, vocabulary verdicts; **native script intact** (ü/ß literal, not `\uXXXX`).

3. **Error paths** (each a clean message, non-zero exit, no misleading artifact):
   - Clear the DB key (`GlobalSettings.GeminiApiKey`) → `--judge gemini` fails fast with "GlobalSettings.GeminiApiKey is not set in the database." *before* any Claude call.
   - A bad model name → the Gemini API error is surfaced (status + body), not a crash.
