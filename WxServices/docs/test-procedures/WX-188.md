# WX-188 ‚ÄĒ Functional test procedure

**Ticket:** WX-188 ‚ÄĒ Extended Forecast grid renders a calendar day whose blocks have all
elapsed at send time ("leads with yesterday").
**Change under test:** `StructuredReportRenderer` now drops a local calendar day from the
Extended Forecast grid once **every** one of its 6-hour blocks has fully elapsed at the
send instant (`nowUtc`), so the grid leads with the day containing the send instant ‚ÄĒ never
a wholly-past "yesterday". Day-level only: a retained day (including the current day) keeps
all its blocks, so its High/Low/wind are unchanged.
**Released in:** 1.30.1 ¬∑ **Component:** WxReport.Svc
**Verify script:** `docs/test-procedures/WX-188-verify.sh`

## Why this needs post-deploy verification

The fix is deterministic and unit-tested (`StructuredReportRendererTests`:
`Grid_DropsWhollyPastDay_LeadsWithCurrentDay`,
`Grid_RetainedDay_HighLowSpanWholeCalendarDay_IncludingElapsedBlocks`,
`Grid_OmittedEntirely_WhenEveryDayIsWhollyPast`). The unit tests are the deterministic proof
of the logic. Production verification confirms the deployed binary renders this way against
**real** snapshots, recipients, timezones, and run timing ‚ÄĒ specifically that a report built
after local midnight on a GFS 00Z run no longer opens its grid with the prior day (the send
2270 repro).

The bug only surfaces in a narrow window ‚ÄĒ a report built in the **early local-morning hours**
(after local midnight, on the 00Z-run snapshot, before later runs shift the snapshot start
forward). The verdict is therefore **preconditioned**: until at least one such report ships
post-deploy, a clean result is `WAIT`, not `PASS`.

## Invariant

For every report delivered since the deploy boundary:

> the **first** Extended-Forecast date row ‚Č• the report's **local build date**
> (`CommittedSends.CreatedAtUtc` converted to the recipient's timezone).

A report whose grid leads with a date *before* its local build date is the failure signature.

## Automated check

```bash
bash docs/test-procedures/WX-188-verify.sh
```

The script resolves the version-pinned deploy boundary (1.30.1 / WxReportSvc) via
`deploy-info.sh`, reads the non-diagnostic `CommittedSends` delivered since, parses each
rendered `EmailBody`'s first grid date, and compares it to the local build date. It reports:

- **reports inspected** ‚ÄĒ the render path ran (the PASS needs ‚Č•1).
- **early-morning reports (00:00‚Äď05:59 local)** ‚ÄĒ the precondition; 0 ‚áí `WAIT`.
- **grid-leads-with-a-past-day violations** ‚ÄĒ the failure signature; expect **0**.

*Expect:* once a full 24 h window has accrued **and** at least one early-morning report has
shipped, with zero violations ‚Üí **PASS** (prints a paste-ready Jira Test Result). Any
violation ‚Üí **FAIL** (lists the offending sends). No early-morning report yet, or window not
yet elapsed ‚Üí **WAIT**.

> Note: the script reads the database via `sqlcmd` (through `powershell.exe`) ‚ÄĒ the first
> verify script to do so, because the fix changes rendered output, which leaves no
> service-log fingerprint. Override the DB target with `WX188_DB_SERVER` / `WX188_DB` if
> needed; the default is `.\SQLEXPRESS` / `WeatherData`.

## Manual spot-check (optional, deterministic backstop)

1. Find an early-morning send delivered after the deploy:
   ```bash
   powershell.exe -NoProfile -Command "sqlcmd -S .\SQLEXPRESS -d WeatherData -E -C -h -1 -W -Q \
     \"SELECT TOP 5 Id, CONVERT(varchar(19),CreatedAtUtc,120), RecipientId FROM CommittedSends \
       WHERE IsDiagnostic=0 AND SentAtUtc IS NOT NULL ORDER BY Id DESC\""
   ```
   Pick one whose `CreatedAtUtc`, in the recipient's local time (America/Chicago = UTC‚ąí5 in
   summer), falls between 00:00 and 06:00.
2. Dump its body and read the first forecast-grid date row and the header date:
   ```bash
   powershell.exe -NoProfile -Command "sqlcmd -S .\SQLEXPRESS -d WeatherData -E -C -y 0 \
     -Q \"SET NOCOUNT ON; SELECT EmailBody FROM CommittedSends WHERE Id=<Id>\""
   ```
   *Expect:* the first row under "Forecast for ‚Ä¶" is the **local build day** (today), not the
   prior day; the header date and the grid's first date agree on the day.
