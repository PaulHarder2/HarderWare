# WX-293 Functional Test Procedure — Closing "storms"-for-a-non-severe-window (post-deployment verification)

**What this verifies:** that WX-293 (1.50.1, the version the verify script pins) is live and that **live Claude complies** with the
strengthened storm-word gate — the reconciler Closing no longer renders "storms" for a non-severe
day-part, so the Closing section is no longer dropped by the WX-189 degrade for this reason. The
fix is **prompt-side** (an absolute, phase-aware storm-word gate + a closing-scoped multi-window
reminder in `ReconcilerPrompts`) plus a **sharpened retry-feedback message** in
`CheckSevereStormVocabulary`. The deterministic backstop is UNCHANGED and still fails closed; the
unit tests prove the prompt text and the backstop, only production proves live Claude complies —
the success state is *silence* (clean reports, no exhausted storm-wording degrades).

**Recording results:** on pass, set the ticket's custom fields — **Test Proc** =
`docs/test-procedures/WX-293.md`, **Test Result** = `PASS yyyy-mm-dd @ <version>`. On failure:
`FAIL yyyy-mm-dd @ <version> — <which check>`, a ticket comment, and open the Bug (the
`FAIL: Open bug ticket` transition).

## 1. KDWH replay fixtures — re-record only if the replay diverges (optional)

WX-293 changed the reconciliation prompt, so per the fixtures README a re-record keeps the KDWH

and a rejection *message* — no validator accept/reject logic — so the recorded Claude response

Re-record is therefore hygiene, not a gate: do it opportunistically at the next golden beat, or

SDK 9; WX-285). Not required for this PASS.

## 2. Confirm the rule is live and Claude complies (the PASS/FAIL signal)

Run the script (step 0) after ≥ one full active day past the `1.50.1` boundary.

- **Expect:** *Reports flowed since deploy* > 0 (the fix is live).
- **Expect:** *EXHAUSTED storm-wording degrades* = **0** — the PASS signal (live Claude renders
  "rain" for non-severe windows and reserves storm language for a severe block).
- *Self-correcting storm-wording retries* ≥ 0 is **fine** — a nudge Claude then obeys is the
  strengthened prompt + sharpened retry feedback working, not a failure.

## 3. If an exhausted storm-wording degrade fires — adjudicate it

The script prints the flagged lines. Read each:

- **Persistent non-compliance** (the prose really did keep "storms" for a non-severe window across
  all 3 retries): live Claude isn't complying → **FAIL**; open a Bug to strengthen the prompt gate
  further (the backstop still protected the recipient by dropping only the offending section).
- **Validator mis-flag** (the window the prose named actually carries a severe block, wrongly
  rejected): → **FAIL**; open a Bug against `CheckSevereStormVocabulary` window-scoping.

## 4. Spot-check a shipped report

Read a recently delivered report whose Closing spans multiple day-parts of a non-severe convective
day. **Expect:** the Closing uses "rain" for every non-severe window (no "storms" varied in for
contrast), and reserves "severe storms" only where a severe block backs it.

**PASS** = reports flowed with zero exhausted storm-wording degrades over a full active day
(step 2); no unadjudicated flagged lines (step 3); shipped report reads clean (step 4). The verify
script's WAIT / PASS / FAIL verdict encodes steps 2–3. (Step 1 KDWH re-record is optional hygiene,
not part of the PASS.)
