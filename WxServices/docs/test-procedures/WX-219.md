# WX-219 — WxManager Translation-QA review tab

Manual UI test procedure. WxManager is a WPF app with no automated UI harness and is **not** CI-built,
so this tab is verified by hand. The judge-package join/discovery logic in `WxServices.Common` is covered
by automated tests (`WxReport.Tests/TranslationQaJudgePackageTests.cs`); this procedure covers the UI.

**Prerequisite:** at least one judge package in `C:\HarderWare\translation-qa\` — a paired
`<iso>.<stamp>.request.json` + `<iso>.<stamp>.judged.json`. The `de.20260627-192255` pair (Gemini-judged)
is the reference fixture used below.

**Build/run:** `dotnet build src\WxManager\WxManager.csproj`, then launch
`C:\HarderWare\BuildCache\WxServices\WxManager\bin\Debug\net8.0-windows\WxManager.exe`.

## Steps

1. Launch WxManager; let the **Setup** tab's prerequisite checks pass.
   *Expect:* the **Translation QA** tab (after Languages) becomes enabled.

2. Click **Translation QA**.
   *Expect:* a single-line header — `Judge package:` selector + **Refresh**, then `Language: …`,
   `Source: …`, a colored **confidence** badge, the judge's note (ellipsized, full text on hover), and
   `Findings: N · Vocab flagged: x/total` on the right. Below it: scenario sub-tabs + a **Vocabulary** tab.

3. Confirm the selector lists each `*.judged.json` that has a sibling `*.request.json`, newest first; pick
   `de · 20260627-192255`.
   *Expect:* trust bar shows `German (de)`, `Source: gemini-2.5-flash …`, a green `confidence: high`
   badge, and the counts.

4. Open a scenario sub-tab (e.g. **winter-frozen**).
   *Expect:* three side-by-side panes — **English reference** | **target (under audit)** | **Back-translation
   → English** — the first two rendered as the recipient email (WebView2), the third the judge's
   back-translation text.

5. In **Report findings**, read through the cards.
   *Expect:* one gold card per location; a location with several findings shows a bulleted list with the
   suggested fix aligned under each problem; the vertical scrollbar reaches the **last** finding's suggested
   fix (nothing cut off at the bottom or behind the taskbar).

6. Select text in a finding (problem or suggested fix) and press **Ctrl+C**, then paste into Notepad.
   *Expect:* the selected text (incl. German characters) copies cleanly.

7. **Maximize** the window.
   *Expect:* the window stops at the top edge of the Windows taskbar (does not extend behind it); the
   findings area remains fully visible.

8. Open the **Vocabulary** tab.
   *Expect:* a table — Status · Token · English · Target · Comment · Suggestion · Apply. Rows with a
   suggestion sort to the top (then alphabetical by token). Status is color-coded
   (green OK / amber Warn / red Wrong / grey not-judged / orange unrepresentable); rows with an actionable
   suggestion show a `✎` and an enabled **Copy → DB** button; other rows' buttons are disabled.

9. (DB-write — use a disposable token.) On a row whose suggestion differs from the target, click
   **Copy → DB**.
   *Expect:* a confirmation dialog showing old → new; on **Yes**, the `LanguageTemplates.Phrase` for that
   `(language, token)` is updated and `ReviewedBy`/`ReviewedAtUtc` stamped; the button reads `applied` and
   disables. (Verify in SSMS if desired.)

10. (Placeholder guard.) If a row's suggestion is missing a `{n}` placeholder the English source has,
    clicking **Copy → DB** must refuse.
    *Expect:* a "Placeholder mismatch" warning; **no** DB write.

11. Click **Refresh**.
    *Expect:* the selected package reloads from disk (picks up any out-of-band change to the files).
