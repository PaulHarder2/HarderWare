# AI Engineering Landscape Scan — July 2026

## TL;DR

Claude Sonnet 5 GA brings ~13% effective input cost savings through Aug 31 vs. Sonnet 4.6, but migrating WxServices requires removing `temperature` from `ClaudeClient.cs` first — Sonnet 5 returns a 400 error on any non-default `temperature` value. After the promo ends Sep 1, Sonnet 5's new tokenizer makes it ~30% more expensive at standard rates: test quality first, then decide whether to stay or revert.

---

## Actionable Findings

### 1. Claude Sonnet 5 — WxServices migration: promo window + breaking code change required

**What changed.** `claude-sonnet-5` went GA. Introductory pricing through August 31, 2026: $2/$10 per MTok (input/output). Standard pricing from September 1: $3/$15 per MTok — identical to Sonnet 4.6.

**Why it matters for Paul's stack.** Two factors interact:

*Tokenizer inflation.* Sonnet 5 uses the newer tokenizer (introduced with Opus 4.7) that produces ~30% more tokens for the same text. This partially offsets the promo discount.

Effective cost comparison on identical input text:
- Sonnet 4.6 today: $3 × 1.0 = **$3.00/MTok effective**
- Sonnet 5 promo (through Aug 31): $2 × 1.3 = **$2.60/MTok effective — ~13% cheaper**
- Sonnet 5 standard (Sep 1+): $3 × 1.3 = **$3.90/MTok effective — ~30% more expensive**

*Breaking API change.* `temperature` set to any non-default value returns HTTP 400 on Sonnet 5 (and Opus 4.7+). `ClaudeClient.cs` sends `temperature` explicitly in both call paths:
- `InvokeReconciliationAsync` (~line 213): `temperature = ReconcilerTemperature` (0.5 — set deliberately in WX-189 to let changeSummary prose "breathe")
- `InvokeTranslationAsync` (~line 405): `temperature = TranslatorTemperature` (0.2 — for faithful translation)

Both parameters must be removed from the request objects before the model ID can be safely changed. Quality impact of removing temperature control is unknown; test on real reconciliation and translation cycles before deploying.

**Cost to add.** Free — API pricing reduction, no new subscription. Net ~13% cheaper on all token classes (input, output, cache writes, cache reads) through Aug 31.

**Recommended next step.**
1. Remove `temperature` from both request objects in `ClaudeClient.cs`.
2. Run real reconciliation and translation cycles and compare output quality against Sonnet 4.6 baseline.
3. If quality is acceptable: update `"Model": "claude-sonnet-4-6"` → `"claude-sonnet-5"` in `appsettings.shared.json` before Aug 31.
4. On Sep 1: reassess. If the quality improvement justifies the ~30% effective cost premium, keep Sonnet 5. Otherwise revert to `claude-sonnet-4-6` (Active; earliest retirement Feb 17, 2027).

### Note: Claude Code default changed to Sonnet 5

Claude Code v2.1.197 (June 30) made `claude-sonnet-5` the default model. The June 2026 scan recommended switching Claude Code from Opus 4.7 to Opus 4.8 for complex agentic coding sessions — that recommendation is unchanged. Sonnet 5 is a reasonable default for general use but Opus 4.8 remains the appropriate tier for long-horizon multi-file WxServices work. No new action; if the June recommendation has not been acted on, it still applies.
