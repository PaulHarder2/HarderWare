# AI Engineering Landscape Scan — May 2026

## TL;DR

Claude Code shipped a `/code-review --comment` plugin that posts multi-agent inline PR comments to GitHub, creating a credible free-at-current-cost alternative to CodeRabbit Pro ahead of the June renewal. Sentry's Seer Agent reached GA with .NET support, surfacing a potential observability gap in WxServices production.

## Actionable Findings

### 1. Claude Code `/code-review --comment` — re-evaluate CodeRabbit Pro before June renewal

**What changed.** The former `/simplify` command was replaced by a proper multi-agent plugin. Running `/code-review --comment` on a PR branch fires 4 parallel analysis agents over the diff, scores each finding by confidence (≥80 posted, lower suppressed), deduplicates, and posts ranked inline comments directly on GitHub PR lines. Each comment includes one-click 👍/👎 rating buttons.

**Why it matters.** Paul pays ~$24/month for CodeRabbit Pro (subscribed April 20; next renewal ~June 20). He already pays for Claude Opus 4.7. This command gives multi-agent inline PR review at no additional cost. The overlap with CodeRabbit's core value proposition is substantial.

**Recommended next step.** Before June 20, run `/code-review --comment` on 2–3 PRs and compare output quality against CodeRabbit's review thread. If findings are comparable, cancel CodeRabbit. Caveat: `/code-review` is triggered manually — CodeRabbit runs automatically on every PR push. If that automation is load-bearing, keep CodeRabbit.

### 2. Sentry Seer Agent GA (April 2026) — check WxServices observability gap

**What changed.** Sentry launched Seer Agent: natural-language, trace-connected investigation of production errors. January 2026 added local-dev and code-review debugging; April 2026 extended it to interactive agent sessions that traverse all connected telemetry. Flat pricing: $40/active contributor/month, unlimited usage.

**Why it matters.** WxServices is a production .NET 8 Windows service stack with no observability layer in the baseline. Runtime errors in long-running services are silent until they become incidents. Sentry's .NET SDK has first-class Windows service support, and Seer Agent lets a developer describe a symptom in plain language and receive traced root causes.

**Recommended next step.** Audit whether WxServices has any production monitoring. If not, start a Sentry free trial and wire the .NET SDK into the service host. Seer Agent is only worth $40/month once telemetry is flowing — borderline for a solo personal project, but worth knowing the gap exists. File a Jira ticket to evaluate.

### 3. Claude Code Agent View + `/goal` command — new parallel-agent workflow pattern

**What changed.** Agent View (research preview, v2.1.139+, released May 11) provides a unified CLI dashboard for all background and foreground sessions. The `/goal` command instructs Claude to work autonomously across turns until a stated completion condition holds.

**Why it matters.** No subscription change required. The pattern of running one background agent generating tests while working interactively in a second session is now first-class in Claude Code. This changes how efficiently solo work can be parallelized on the existing Max plan.

**Recommended next step.** Update Claude Code, then try: kick off a background session with `/goal "write unit tests for WxServices.WeatherPoller with full branch coverage"`, switch to a foreground session for active coding, and monitor both from `claude agents`. This is a new working pattern, not a new tool.
